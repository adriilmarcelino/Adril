01. Pengertian OOP

Object Oriented Programming (OOP) Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek, dimana program dibuat menggunakan class dan objek. Dalam OOP terdapat beberapa konsep utama seperti Encapsulation, Inheritance, Polymorphism, dan Abstraction yang membantu membuat program lebih terstruktur, modular, dan mudah dikembangkan.

02. Konsep Dasar

  • Class: Blueprint atau cetakan untuk membuat objek.
  • Object: Instance nyata yang tercipta dari sebuah class.
  • Attribute: Representasi data melalui variabel di dalam class.
  • Method: Logika fungsional atau fungsi di dalam class.

03. Class Sederhana (Drone)

Pada contoh program di bawah ini terdapat class Drone yang memiliki atribut model dan jumlahBalingBaling untuk menyimpan data teknis perangkat. Terdapat pula method terbang() dan mendarat() sebagai aksi utama dari objek drone tersebut.

class Drone {
  String? model;
  int? jumlahBalingBaling;
  
  void terbang() {
    print('Drone $model sedang lepas landas...');

07. Encapsulation

Enkapsulasi melindungi integritas data dengan menyembunyikan variabel _poin menggunakan akses privat. Poin hanya bisa dimodifikasi melalui method tambahPoin() dan dibaca melalui getter cekPoin untu menjaga keamanan data member.

DART SOURCE CODE
// Nama   : Adriel nofra marcelino
// Nim    : 251410017
// Kelas  : SI2A

class Member {
  String nama;
  int _poin; 

  Member(this.nama, this._poin);

  void tambahPoin(int jumlah) {
    if (jumlah > 0) _poin += jumlah;
  }
  
  int get cekPoin => _poin;
}

void main() {
  Member memberAdriel = Member("Adriel Nofra", 100);
  print('Member: ${memberAdriel.nama}'); 
  memberAdriel.tambahPoin(50);
  print('Total Poin: ${memberAdriel.cekPoin}'); 
}

08. This Keyword

Keyword "this" digunakan sebagai referensi ke atribut di dalam class itu sendiri. Pada contoh class Sistem, this.versi digunakan agar komputer tahu bahwa yang dimaksud adalah variabel milik class, bukan parameter fungsi.

09. Product Class (Digital)

Class ProdukDigital menggunakan "Named Parameters" untuk memudahkan pemetaan data saat pembuatan objek. Dengan fitur ini, kita bisa memberikan argumen berdasarkan nama filenya, yang membuat kode lebih mudah dibaca dan dipelihara.

DART SOURCE CODE
// Nama   : Adriel nofra marcelino
// Nim    : 251410017
// Kelas  : SI2A

class ProdukDigital {
  final String nama, lisensi; 
  final double harga; 

  ProdukDigital({
    required this.nama,
    required this.harga,
    this.lisensi = 'Single User',
  });
}

void main() {
  ProdukDigital item1 = ProdukDigital(
    nama: "UI Design Pack",
    harga: 250000,
    lisensi: "Commercial",
  );

  ProdukDigital item2 = ProdukDigital(
    nama: "Asset 3D Model",
    harga: 150000,
  );

  print('Produk 1: ${item1.nama} (Lisensi: ${item1.lisensi})'); 
  print('Produk 2: ${item2.nama}'); 
}
